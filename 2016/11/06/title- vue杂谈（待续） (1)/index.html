什么是vue.js？

vue.js(以下简称为vue)是一种轻量高效的组件化开发方式。简单来说就是将页面的各个部分单独编写成组件，再在需要使用的地方将其渲染出来，以达到减轻代码编写量、提高编程效率的功效。在本次项目中我主要使用了vue中的“class和style绑定”、“条件渲染”、“列表渲染”和“组件”等知识。下面我将简单讲讲我第一次使用遇到的困惑或难点、常用点。

组件的复用

在“晓阅”项目中，header几乎出现在所有页面，除了文字和颜色的变化之外，其他样式是一样的。这时就要充分发挥vue的功用，通过传递参数实现复用。代码如下：

<div class="header-padding">

    <div class="headerConatiner" :style="{ backgroundColor: headerBgcolor, color: headerTextcolor }">

        <div class="flex-header-left" v-on:click="leftOnClick"><span></span></div>

        <div class="flex-header-text">{{ headerText }}</div>

        <div class="flex-header-right" v-on:click="rightOnClick">{{ rightText }}</div>

    </div>

</div>

当调用时只需传入参数，如：

<reg-header header-bgcolor="rgb(90,86,87)" header-textcolor="#E7E7E7" header-text="我要打卡"></reg-header>

如此一来就能重复调用header组件，即使内容不同，传个参数就能解决。

vue-router

vue-router初探

vue-router在实现路由跳转方面的使用十分方便，但是其中的逻辑问题一旦弄混就容易出现bug。在我们使用的时候必须先理清整个路由跳转的思路，比如“这个组件是在哪个组件内渲染的？”、“链接跳转时路由的变化。”等。下面我就来讲一些我遇到的问题。

vue-router易错点

我有一个app.vue文件，一个homepage.vue文件，一个center.vue文件和footer.vue文件。一开始我是这样写的：

var index=app.vue;
var homepage=../../homepage.vue;
var center=../../center.vue;
router.map({
    "/": {
        name: 'root',
        component: index
        }，
    "/homepage": {
        name: "homepage",
        component: homepage
    },
    "/center": {
        name: "center",
        component: center
    }
}

如果我将footer写进app.vue里面，那么上面这段代码将会产生意想不到的效果：
我们打开网页时先会看到footer，点击footer里的homepage后，footer就会消失不见。这是为什么？
因为在我把”/“和”/homepage”写成了兄弟关系，当我点击跳转到homepage组件时，就会渲染出组件里的内容并将app.vue覆盖掉。
实际上app.vue的作用应该是提供一个地方让homepage等组件可以渲染，同时保证footer不被覆盖。
实际上这几个文件的关系如下：
因此代码应改为：

"/": {
        name: 'root',
        component: index,
        subRoutes: {
            "/" : {
                name : 'homepage',
                component: homepage
                },
            "/center": {
                name: "center",
                component: center
                }
        }
    }

我是这样理解路由跟组件的关系的：

  1. 当两个组件对于第三个组件来说都是子代关系时，他们在“父组件”中的渲染可以看成里内容的替换关系，而“父组件”外的内容均不发生改变。
  2. 当两个组件对于第三个组件来说都是兄弟关系时，他们是直接切换组件的渲染，他们之间不存在作为共同渲染容器。
