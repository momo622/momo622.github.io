JS引用类型之RegExp类型

发表于 2016-08-01

标签：JavaScript
– 该部分是我学习引用类型时做的笔记，下面直入主题。

正则表达式规则

ECMAScript通过RegExp类型来支持正则表达式，可以通过以下途径创建：

var example = / pattern / flags ;

pattern可以是任何形式的正则表达式，而flags则用以表明正则表达式的行为。flags由以下三种形式

  • g：表示global，即模式将被用于所有字符串，并不因为寻找到了匹配项而停止。
  • i：表示case-insensitive，对大小写不敏感。

  • m：表示多行模式，即会自动换行寻找匹配项。

    var pattern1  =/\[bc\]at/i；//表示匹配第一个“[bc]at”,不区分大小写。

var pattern2 =/.at/gi ;   //表示匹配所有以at为结尾的字符组合，不区分大小写。

var pattern3 = /[bc]at/g ;    //表示匹配所有“bat”或“cat”。

同时还可以使用构造函数创建：

如var pattern4 = new RegExp("[bc]at","i");//这与pattern1完全相等。

但要注意，不能把正则表达式字面量传递给构造函数。由于RegExp构造函数的模式参数是字符串，所以某些情况下需要双重转义。

字面量模式 等价字符串
/\[bc\]at/ "\\[bc\\]at"
/\.at/ "\\.at"
/name\/gae/ "name\\/age"
/\d.\d{1,2}/ "\\a.\\d{1,2}"

使用正则表达式字面量与使用RegExp构造函数创建正则表达式的区别

JavaScript高程中提到，在ECMAScript3中，正则表达式字面量始终共用一个实例，而后者则每次都创建一个新实例。

下面来看书上的例子：

var re = null ,
i;

for ( i=0 ;i < 10 ;i++)
{
    re=/cat/g;
    re.test("catastrophe");
}

for ( i=0 ;i < 10 ;i++ )
{
    re = new RegExp("cat","g");
    re.test("catastrophe");
}

第一个循环共用一个实例，所以第一次匹配到cat后，第二次是从上一次匹配的末尾开始，所以返回false。而第二个循环每次新建一个实例，每次都可以找到“cat”，故返回true。
而ECMAScript5规定使用正则表达式字面量必须每次都创建一个实例。

RegExp实例属性

  • global：布尔值，表示是否设置了g标志。
  • ignoreCase:布尔值，表示是否设置了i标志。
  • lastIndex:表示开始搜索下一个匹配项的字符位置，从0算起。
  • multiline：布尔值，表示是否设置了m属性。
  • source：正则表达式的字符串表示（按照字面量形式返回）。

RegExp实例方法

exec( )

该方法主要用于捕获组。exec（）接受一个字符串，然后返回包含第一个匹配信息的数组，或返回null。其中还包含两个属性，index（匹配项在字符串中的位置）和input（应用正则表达式的字符串）

var text ="cat,bat,sat,fat";
var pattern1 = /.at/;
var matches = pattern1.exec(text);
alert(matches.index);//0
alert(matches[0]);//cat
alert(pattern1.lastIndex);//0

在全局模式下，lastIndex每次调用exec（）后都会改变，而非全局则不会。（IE浏览器除外）。

text（）

另外还有text（）方法，他接受字符串参数，然后判断他与某个模式是否匹配，返回true or false。

其他

toLocalString（）和toString（）均返回正则表达式字面量。
valueOf则返回正则表达式本身。

谈谈css页面布局之浮动

发表于 2016-07-17

标签： css 浮动与定位

—阅读《css权威指南》后对浮动有了一些更细致的了解，以下是我的读书笔记和个人理解。

过去，只可能浮动图像（某些浏览器可能还支持表的浮动）。但css允许浮动任何元素，从图像到段落再到列表，所有元素都可以浮动。在css中，这种行为使用属性float实现。 ————《css权威指南》

何为浮动

float 属性定义元素在哪个方向浮动。浮动元素会生成一个块级框，而不论其本身是哪种元素。float属性可能的值有left，right，inherit，none。

浮动层

float属性使元素脱离文本流，其外边距紧贴于父元素内边距进行浮动。
而此浮动元素在文档流空出的位置，由后续的(非浮动)元素填充上去。

浮动特点

浮动元素几乎自成一派，但会对文档剩余部分有影响。一个元素浮动时，其他内容会环绕该元素。另外，无论该元素本身是什么，浮动元素都会生成一个块级框。即使原本是行内元素，通常都会生成一个行内框。

浮动元素外边框不合并

如图所示两个正方向均有20px的外边距，但红色方块上边距与黑色方块下边距并不会合并，而是变成40px。

浮动的规则

1.浮动元素的左（右）外边界不能超出其包含块的左（右）内边界
2.为了避免产生重叠，浮动元素的左（右）外边界必须是先前出现的浮动元素的右（左）外边界。后出现的浮动元素的顶端在先出现元素的底部这种情况也是合理的。这是浮动不同于定位的一点，这样一来所有的浮动内容都是可见的。
3.浮动元素不能超过其包含元素的边界，除非他本身就宽于包含元素。如图1所示，在width为450px的容器中，两个200px的方块加上20px的外边距显然会超出边界。这是浮动元素会被挤到新一行。
4.一个浮动元素的顶端不能比之前所有浮动元素或块级元素顶端高。

<div class="black"> </div>
<div class="red"> </div>

在代码行中黑色方块的出现先于2红色方块，这就是为什么换行时红色方块下落。同样的，交换书写顺序将使黑块下落。
5.行内框和浮动元素重叠，前者的背景，内容等都将显示在后者之上。而块框元素与之重叠时，背景在后者之下，内容在后者之上。

负边距与浮动

使用负边距可以使浮动元素移出其父元素。乍一看好像觉得与前面所说的相矛盾，但仔细想想，当外边界是负值时，相当于父元素的外边界贴于浮动元素内，因此移出父元素属于合理想象。

clear的使用

有时我们不想某些内容某个方向出现浮动元素，这时可以借助clear去解决。例如我想让浮动的图片呈竖向排列，就可以img{float:left;clear:both;}
ps:clear的值有none，both，inherit，left，right五种。
