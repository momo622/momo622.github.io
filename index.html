什么是vue.js？

vue.js(以下简称为vue)是一种轻量高效的组件化开发方式。简单来说就是将页面的各个部分单独编写成组件，再在需要使用的地方将其渲染出来，以达到减轻代码编写量、提高编程效率的功效。在本次项目中我主要使用了vue中的“class和style绑定”、“条件渲染”、“列表渲染”和“组件”等知识。下面我将简单讲讲我第一次使用遇到的困惑或难点、常用点。

组件的复用

在“晓阅”项目中，header几乎出现在所有页面，除了文字和颜色的变化之外，其他样式是一样的。这时就要充分发挥vue的功用，通过传递参数实现复用。代码如下：

<div class="header-padding">

    <div class="headerConatiner" :style="{ backgroundColor: headerBgcolor, color: headerTextcolor }">

        <div class="flex-header-left" v-on:click="leftOnClick"><span></span></div>

        <div class="flex-header-text">{{ headerText }}</div>

        <div class="flex-header-right" v-on:click="rightOnClick">{{ rightText }}</div>

    </div>

</div>

当调用时只需传入参数，如：

<reg-header header-bgcolor="rgb(90,86,87)" header-textcolor="#E7E7E7" header-text="我要打卡"></reg-header>

如此一来就能重复调用header组件，即使内容不同，传个参数就能解决。

vue-router

vue-router初探

vue-router在实现路由跳转方面的使用十分方便，但是其中的逻辑问题一旦弄混就容易出现bug。在我们使用的时候必须先理清整个路由跳转的思路，比如“这个组件是在哪个组件内渲染的？”、“链接跳转时路由的变化。”等。下面我就来讲一些我遇到的问题。

vue-router易错点

我有一个app.vue文件，一个homepage.vue文件，一个center.vue文件和footer.vue文件。一开始我是这样写的：

var index=app.vue;
var homepage=../../homepage.vue;
var center=../../center.vue;
router.map({
    "/": {
        name: 'root',
        component: index
        }，
    "/homepage": {
        name: "homepage",
        component: homepage
    },
    "/center": {
        name: "center",
        component: center
    }
}

我们打开网页时先会看到footer，点击footer里的homepage后，footer就会消失不见。这是为什么？
因为在我把”/“和”/homepage”写成了兄弟关系，当我点击跳转到homepage组件时，就会渲染出组件里的内容并将app.vue覆盖掉。
实际上app.vue的作用应该是提供一个地方让homepage等组件可以渲染，同时保证footer不被覆盖。
实际上这几个文件的关系如下：
因此代码应改为：

"/": {
        name: 'root',
        component: index,
        subRoutes: {
            "/" : {
                name : 'homepage',
                component: homepage
                },
            "/center": {
                name: "center",
                component: center
                }
        }
    }

我是这样理解路由跟组件的关系的：

  1. 当两个组件对于第三个组件来说都是子代关系时，他们在“父组件”中的渲染可以看成里内容的替换关系，而“父组件”外的内容均不发生改变。
  2. 当两个组件对于第三个组件来说都是兄弟关系时，他们是直接切换组件的渲染，他们之间不存在作为共同渲染容器。

协同主页开发挖坑记录

发表于 2016-08-24

标签： 项目开发
– 世上本没有坑，挖的人多了便也成了坑。

项目简要总结

我们在正式做项目前讨论了整个项目的思路，决定在项目中运用一些插件。所以我先研究了Bootstrap时间日期选择器的插件和栅格布局。恰好刚学习了Canvas的用法，而且我所负责的部分是各组组别，于是想试用一下Canvas。于是，一个不明就里的大坑由此出现。

Canvas与栅格布局的大碰撞

我的思路是用Canvas画两个方块，蓝色的叠加在橙色之上再用jQuery中的hide()函数控制蓝色的透明度，使颜色渐变成橙色。由于我是先写好了Canvas才考虑使用栅格布局，所以情况就变得有点出乎意料。原本使用栅格布局可以轻松完成的自适应，到了实际调试时发现缩小页面会将蓝色的方块挤出文本流。原因就是栅格布局中margin和padding的问题。我尝试用一个div包住Canvas然后改变其样式，这确实是行之有效的，但是违背了栅格布局方便简单的原则。在多次尝试无果之后，我改变了思路重新编写这一部分。新的问题就发生了。

透明度继承问题

<div class="col-lg-4 col-md-6 col-xs-6 grpBlock" id="fe">
    <div class="backgroundOrange">
        <div class="backgroundBlue">
        <img src="images/fe.png" width="130" height="130" />
        </div>
    </div>    
</div>

在这种情况下就会出现鼠标移至蓝色方块上时白色图标跟着一起消失，因为img继承了父元素的透明度。强行在css中固定图标的透明度或者设置z-index都是没有效果的。如果让img跟backgroundBlue成为兄弟元素则会改变橙色背景的大小。看了网上一些解决透明度继承的方法于是改变了一下代码。

<div class="col-lg-4 col-md-6 col-xs-6 grpblock" id="fe">
    <div>
        <div class="backgroundorange">
            <div class="backgroundblue"></div>
        </div>    
        <img src="images/fe.png" width="130" height="130" />
    </div>                
</div>

然后设置img的position为relative，再调整margin就能让img始终在父元素中居中。这里总结解决透明度继承的一个方法就是让他们从父子关系变成兄弟关系，具体情况具体分析。
除此之外还有很多各式各样的问题，以后再一一总结。

反思

在这次项目中走弯路很大程度上是因为我并没有事先考虑整个框架。同时因为经验不足，总是在编写代码中才不断发现错误、发现新的思路，新的思路和技术或许又与原来的框架有所矛盾。另外我花了两天时间在修改同一个问题都没有得到很好的解决，尔后转换思路重写编写只花了半小时。所以以后当问题长时间都没有得到解决时可以放下原有的思路，不要长时间纠结于更改代码，除非你对原有的思路很有信心。

感谢

感谢在代码的编写过程中小伙伴们给予的帮助，希望在以后的日子里我们也可以互帮互助、共同进步！

JS引用类型之RegExp类型

发表于 2016-08-01

标签：JavaScript
– 该部分是我学习引用类型时做的笔记，下面直入主题。

正则表达式规则

ECMAScript通过RegExp类型来支持正则表达式，可以通过以下途径创建：

var example = / pattern / flags ;

pattern可以是任何形式的正则表达式，而flags则用以表明正则表达式的行为。flags由以下三种形式

  • g：表示global，即模式将被用于所有字符串，并不因为寻找到了匹配项而停止。
  • i：表示case-insensitive，对大小写不敏感。

  • m：表示多行模式，即会自动换行寻找匹配项。

    var pattern1  =/\[bc\]at/i；//表示匹配第一个“[bc]at”,不区分大小写。

var pattern2 =/.at/gi ;   //表示匹配所有以at为结尾的字符组合，不区分大小写。

var pattern3 = /[bc]at/g ;    //表示匹配所有“bat”或“cat”。

同时还可以使用构造函数创建：

如var pattern4 = new RegExp("[bc]at","i");//这与pattern1完全相等。

但要注意，不能把正则表达式字面量传递给构造函数。由于RegExp构造函数的模式参数是字符串，所以某些情况下需要双重转义。

字面量模式 等价字符串
/\[bc\]at/ "\\[bc\\]at"
/\.at/ "\\.at"
/name\/gae/ "name\\/age"
/\d.\d{1,2}/ "\\a.\\d{1,2}"

使用正则表达式字面量与使用RegExp构造函数创建正则表达式的区别

JavaScript高程中提到，在ECMAScript3中，正则表达式字面量始终共用一个实例，而后者则每次都创建一个新实例。

下面来看书上的例子：

var re = null ,
i;

for ( i=0 ;i < 10 ;i++)
{
    re=/cat/g;
    re.test("catastrophe");
}

for ( i=0 ;i < 10 ;i++ )
{
    re = new RegExp("cat","g");
    re.test("catastrophe");
}

第一个循环共用一个实例，所以第一次匹配到cat后，第二次是从上一次匹配的末尾开始，所以返回false。而第二个循环每次新建一个实例，每次都可以找到“cat”，故返回true。
而ECMAScript5规定使用正则表达式字面量必须每次都创建一个实例。

RegExp实例属性

  • global：布尔值，表示是否设置了g标志。
  • ignoreCase:布尔值，表示是否设置了i标志。
  • lastIndex:表示开始搜索下一个匹配项的字符位置，从0算起。
  • multiline：布尔值，表示是否设置了m属性。
  • source：正则表达式的字符串表示（按照字面量形式返回）。

RegExp实例方法

exec( )

该方法主要用于捕获组。exec（）接受一个字符串，然后返回包含第一个匹配信息的数组，或返回null。其中还包含两个属性，index（匹配项在字符串中的位置）和input（应用正则表达式的字符串）

var text ="cat,bat,sat,fat";
var pattern1 = /.at/;
var matches = pattern1.exec(text);
alert(matches.index);//0
alert(matches[0]);//cat
alert(pattern1.lastIndex);//0

在全局模式下，lastIndex每次调用exec（）后都会改变，而非全局则不会。（IE浏览器除外）。

text（）

另外还有text（）方法，他接受字符串参数，然后判断他与某个模式是否匹配，返回true or false。

其他

toLocalString（）和toString（）均返回正则表达式字面量。
valueOf则返回正则表达式本身。

谈谈css页面布局之浮动

发表于 2016-07-17

标签： css 浮动与定位

—阅读《css权威指南》后对浮动有了一些更细致的了解，以下是我的读书笔记和个人理解。

过去，只可能浮动图像（某些浏览器可能还支持表的浮动）。但css允许浮动任何元素，从图像到段落再到列表，所有元素都可以浮动。在css中，这种行为使用属性float实现。 ————《css权威指南》

何为浮动

float 属性定义元素在哪个方向浮动。浮动元素会生成一个块级框，而不论其本身是哪种元素。float属性可能的值有left，right，inherit，none。

浮动层

float属性使元素脱离文本流，其外边距紧贴于父元素内边距进行浮动。
而此浮动元素在文档流空出的位置，由后续的(非浮动)元素填充上去。

浮动特点

浮动元素几乎自成一派，但会对文档剩余部分有影响。一个元素浮动时，其他内容会环绕该元素。另外，无论该元素本身是什么，浮动元素都会生成一个块级框。即使原本是行内元素，通常都会生成一个行内框。

浮动元素外边框不合并

如图所示两个正方向均有20px的外边距，但红色方块上边距与黑色方块下边距并不会合并，而是变成40px。

浮动的规则

1.浮动元素的左（右）外边界不能超出其包含块的左（右）内边界
2.为了避免产生重叠，浮动元素的左（右）外边界必须是先前出现的浮动元素的右（左）外边界。后出现的浮动元素的顶端在先出现元素的底部这种情况也是合理的。这是浮动不同于定位的一点，这样一来所有的浮动内容都是可见的。
3.浮动元素不能超过其包含元素的边界，除非他本身就宽于包含元素。如图1所示，在width为450px的容器中，两个200px的方块加上20px的外边距显然会超出边界。这是浮动元素会被挤到新一行。
4.一个浮动元素的顶端不能比之前所有浮动元素或块级元素顶端高。

<div class="black"> </div>
<div class="red"> </div>

在代码行中黑色方块的出现先于2红色方块，这就是为什么换行时红色方块下落。同样的，交换书写顺序将使黑块下落。
5.行内框和浮动元素重叠，前者的背景，内容等都将显示在后者之上。而块框元素与之重叠时，背景在后者之下，内容在后者之上。

负边距与浮动

使用负边距可以使浮动元素移出其父元素。乍一看好像觉得与前面所说的相矛盾，但仔细想想，当外边界是负值时，相当于父元素的外边界贴于浮动元素内，因此移出父元素属于合理想象。

clear的使用

有时我们不想某些内容某个方向出现浮动元素，这时可以借助clear去解决。例如我想让浮动的图片呈竖向排列，就可以img{float:left;clear:both;}
ps:clear的值有none，both，inherit，left，right五种。
